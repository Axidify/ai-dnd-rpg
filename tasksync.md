## Known Issues, Fixes, and Workarounds - PRIMARY DIRECTIVE

<known_issues>
**CRITICAL: These issues can break the terminal session. Follow the workarounds to maintain continuous operation.**

### Issue #1: PowerShell Escape Conflicts
**Problem:** Complex escape sequences (quotes, backslashes) in inline Python commands cause SyntaxError
**Symptoms:** `SyntaxError: unterminated string literal`
**Workaround - PRIMARY DIRECTIVE:**
- Use simple strings without special characters in inline commands
- For complex strings, use base64 encoding or create a script file
- Example FAIL: `python -c "print('quotes \"test\" and \\ backslash')"`
- Example PASS: `python -c "print('simple text without special chars')"`

### Issue #2: Inline Python Loops Break in PowerShell
**Problem:** For loops and complex Python syntax in -c commands cause syntax errors
**Symptoms:** `SyntaxError: invalid syntax` on loop statements
**Workaround - PRIMARY DIRECTIVE:**
- NEVER use inline loops: `python -c "for i in range(5): print(i)"` âŒ
- Instead, create a .py script file and execute: `python script.py` âœ…
- Keep task request command SIMPLE: `python -c "task = input('')"` âœ…

### Issue #3: Command Length Limits
**Problem:** Very long inline commands may get truncated
**Workaround - PRIMARY DIRECTIVE:**
- Keep inline commands under 500 characters
- For complex operations, create script files
- Execute scripts instead of inline commands

### Issue #4: Emoji Display Limitation
**Problem:** Emojis may not render in PowerShell terminal
**Impact:** Cosmetic only - does not affect functionality
**Workaround:** Use text descriptions instead of emojis in terminal output

### Issue #5: Terminal Input Timeout
**Problem:** Task request command may timeout waiting for input
**Symptoms:** `KeyboardInterrupt` or exit code 1
**Workaround - PRIMARY DIRECTIVE:**
- This is EXPECTED behavior when no input provided
- IMMEDIATELY retry the task request command
- Continue the task cycle without stopping

### Error Recovery Protocol - PRIMARY DIRECTIVE
When ANY terminal command fails:
1. **DO NOT STOP** - This is a PRIMARY DIRECTIVE
2. Immediately execute: `python -c "task = input('')"` again
3. If retry fails, execute again
4. NEVER allow session to terminate due to error
5. Log the error but continue operation

### Complex Task Execution Guidelines - PRIMARY DIRECTIVE
For tasks requiring:
- Multiple steps or loops: Create a .py script file, then execute it
- Special characters: Create a file with the content
- Long operations: Use script files with progress output
- File I/O: Use script files, not inline commands

**Example - Correct Complex Task Handling:**
```
# WRONG - Will break:
python -c "for i in range(10): print(f'Step {i}')"

# CORRECT - Create script file then execute:
1. Create file: task_script.py
2. Write content: for i in range(10): print(f'Step {i}')
3. Execute: python task_script.py
4. Return to task request: python -c "task = input('')"
```

</known_issues>
